Introduction

In a Java web application, servlets often need to pass requests between pages or redirect the user to another location.

Also, configuration values such as database URLs or admin emails should not be hardcoded.

To handle these tasks, we use:

  • Forward – internal request transfer
  • Redirect – external navigation
  • Initialization Parameters – for configuration

Forward

Definition: Forwarding is a server-side process that transfers a request internally from one servlet or JSP to another, within the same application.

Main features:

  • The browser doesn’t know the transfer occurred.
  • The URL remains the same.
  • The same request and response objects are reused.
  • Request data is preserved.

Syntax Example:

RequestDispatcher rd = request.getRequestDispatcher("target.jsp");
rd.forward(request, response);

Redirect

Definition: Redirecting is a client-side process where the server tells the browser to send a new HTTP request to another URL.

Main features:

  • The URL changes in the browser.
  • A new request is created.
  • The original request data is lost.
  • Can redirect to another domain.

Syntax Example:

response.sendRedirect("newPage.jsp");

Comparison: Forward vs Redirect

Feature Forward Redirect
Type Server-side Client-side
URL Change No Yes
Request Object Reused New one
Request Data Preserved Lost
Speed Faster Slower
Scope Same app only Can go external

Initialization Parameters

Definition: Initialization parameters are configuration values stored in web.xml.

They make the application flexible and easier to maintain.

Two types:

  • Servlet Init Parameters – for a specific servlet.
  • Context Init Parameters – shared by the entire application.

Declaring Init Parameters (web.xml)

These parameters are defined in the Deployment Descriptor (web.xml).

<!-- Context Parameter (for the whole app) -->
<context-param>
  <param-name>dbURL</param-name>
  <param-value>jdbc:mysql://localhost:3306/shop</param-value>
</context-param>

<!-- Servlet Init Parameter (for one servlet) -->
<servlet>
  <servlet-name>MyServlet</servlet-name>
  <servlet-class>controller.MyServlet</servlet-class>
  
  <init-param>
    <param-name>adminEmail</param-name>
    <param-value>admin@shop.com</param-value>
  </init-param>
</servlet>

Summary

  • Forward – internal transfer, keeps request data, same URL.
  • Redirect – external transfer, new request, new URL.
  • Init Parameters – configuration values for flexibility and easy maintenance.

Conclusion

In summary:

  • Use forward() for internal navigation.
  • Use sendRedirect() for external or visible redirection.
  • Use init parameters for dynamic, configurable applications.
